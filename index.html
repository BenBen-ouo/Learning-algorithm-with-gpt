YOLO 演算法介紹與 GPT 輔助學習

111316033 謝百泓

這份網頁筆記是我在學習 YOLO 物件偵測演算法時整理出來的內容。 一開始我只大概知道 YOLO 可以把圖片中的物體框出來，但對於 grid cell、 bounding box、confidence score、NMS 和 mAP 這些名詞其實沒有很完整的概念。 所以我主要透過網路文章、他人的筆記整理，再搭配 GPT 協助釐清卡住的地方， 最後把我理解後的內容整理成這份自學紀錄。

Task 01 · 演算法介紹

YOLO 簡介

YOLO，全名是 You Only Look Once，是物件偵測領域中很常被介紹到的演算法。 它不是只判斷圖片裡「有什麼東西」，而是要同時回答物體的類別、位置，以及模型對這個判斷的信心程度。

它主要在做什麼？

YOLO 的工作是 object detection，也就是在一張圖片或影片畫面中找出多個物體， 並用 bounding box 把物體位置框起來。每個框通常會搭配類別名稱與 confidence score， 例如「person 0.91」、「car 0.86」。因此它和一般 image classification 不同， 因為分類模型通常只輸出整張圖片的類別，而 YOLO 還要指出物體在畫面中的位置。

它解決的問題

在實際影像任務裡，我們常常不只想知道畫面中有沒有某種物體， 而是想知道物體出現在哪裡、數量有多少、是否重疊、是否需要即時反應。 YOLO 的價值就在於把定位與分類整合在同一個模型流程中， 讓物件偵測可以更接近即時應用，例如交通監控、醫學影像輔助判讀、工業檢測和無人機巡檢。

常見應用

YOLO 的特色是能在速度與偵測效果之間取得不錯的平衡，因此很適合需要快速分析影像的情境。 不同應用會重視不同指標，例如交通場景可能重視即時性，醫學或工業檢測則會更在意漏判與誤判的成本。

  • 自動駕駛
  • 監視器辨識
  • 醫學影像
  • 工業瑕疵檢測
  • 農業辨識
  • 無人機巡檢
Task 02 · GPT 輔助學習紀錄

使用 GPT 輔助我學習 YOLO

只用網路上的資源學習仍然有部分不理解的地方，搭配 GPT 的輔助，釐清我不熟悉的內容。

我問 GPT：

我看文章時知道 YOLO 會把圖片切成 grid cell，但我不太懂： 如果一個物體橫跨好幾格，到底是哪一格要負責預測這個物體？

GPT 回答節錄：

YOLO 的概念不是讓物體碰到的每一格都負責，而是看物體中心點落在哪一個 grid cell。 中心點所在的那格會負責預測這個物體的 bounding box。這樣可以避免每個物體被太多格同時負責， 但如果物體很小、很密集或中心點很接近，就可能造成偵測困難。

我問 GPT：

YOLO 的 confidence score 是不是就等於「這個框裡面有某個類別」的機率？ 還是它其實跟 bounding box 的準確度也有關？

GPT 回答節錄：

在 YOLOv1 的概念中，confidence score 不只是單純的類別機率， 它包含「框中是否有物體」以及「預測框和真實框重疊程度」的概念。 所以它和 IoU 有關。真正判斷類別時，還會再結合 class probability， 得到每個類別對應的分數。

Task 03 · 查證與修正

查證 GPT 輔助我的內容是否正確

GPT 在解釋 confidence score 時有提到它和 IoU 有關，但我一開始覺得有點矛盾： 如果 confidence 是模型對類別的信心，那為什麼又會牽涉到預測框和真實框的重疊程度？ 因為這個地方會影響我後面理解 mAP、NMS 和偵測結果的判讀，所以我另外搜尋文章確認。

3.1 GPT 提供的說法

GPT 說 YOLO 的 confidence score 不是單純的分類機率，而是和「是否有物體」以及 「bounding box 預測得準不準」有關。也就是說，一個框即使類別看起來像某個物體， 但如果框的位置和大小不準，最後的分數仍然可能受到影響。

3.2 修正前理解

我原本把 confidence score 想成「模型覺得它是某類物體的機率」， 所以會把它和 class probability 混在一起。這樣理解會造成一個問題： 我會忽略 bounding box 本身的準確度，只注意類別名稱和數字高低。

3.3 搜尋更多資料後得到的內容

我查到 YOLOv1 的說明中，confidence 和 bounding box 與 ground truth 的 IoU 有關； 而每個 grid cell 還會預測類別的條件機率。測試階段會把 bounding box 的 confidence 和 class probability 結合，得到 class-specific confidence score。 因此，confidence、class probability、IoU 其實是相關但不同的概念。

3.4 修正後理解

我後來比較能理解 YOLO 的輸出不是只有「這是什麼類別」， 而是同時包含「哪裡有物體」、「框得準不準」、「它屬於哪一類」。 這也讓我理解為什麼物件偵測不能只用 accuracy 判斷好壞，而需要看 IoU、precision、 recall、AP 和 mAP 這類指標。

這一段查證讓我發現，GPT 比較適合用來幫我把卡住的概念先講順， 但只看 GPT 的回答還不夠。因為 YOLO 的名詞彼此很接近，像 confidence score、 class probability、IoU 和 mAP，如果沒有回去對照筆記文章或論文架構，很容易以為自己懂了， 但其實只是把幾個名詞連在一起。

透過這次比對資料，我比較能用自己的方式理解：YOLO 的輸出是一組「位置 + 類別 + 信心」的組合， 而不是單一的分類答案。這個理解也銜接到後面流程圖的部分，因為整個流程從影像輸入、 CNN 特徵擷取、grid cell 預測、bounding box 評分，到最後 NMS 篩選，其實都是為了把這組結果整理出來。

Task 04 · 視覺化說明

YOLO 流程圖與詳細說明

這一節放入 YOLO 的流程圖，並用我自己的理解拆解圖中的每個階段。

圖片建議使用 Tommy Huang Medium 文章〈深度學習-物件偵測: You Only Look Once (YOLO)〉中， 「YOLO物件偵測怎麼做的」段落下方那張說明 grid cell、bounding box、class probability 和 final detection 的圖。

  • 1. 輸入影像與 resize YOLO 會先把輸入圖片調整成固定尺寸，讓模型可以用一致的格式處理影像。 這個步驟看似只是調整大小，但其實會影響後面物體框的位置比例， 所以實際使用時通常也要注意影像解析度和物體大小。
  • 2. CNN 擷取特徵 接著影像會進入卷積神經網路。CNN 會從低階特徵開始擷取，例如邊緣、線條、顏色變化， 再逐漸形成較高階的形狀與物體特徵。YOLO 的速度優勢來自它直接用一個網路流程處理整張圖， 不需要先另外產生大量候選區域。
  • 3. Grid cell 負責預測 YOLO 會把影像切成 S × S 的 grid。當某個物體的中心點落在某一格， 那一格就負責預測該物體。這個設定讓每個物體有明確的負責區域， 但也解釋了為什麼 YOLO 在處理很小、很密集或彼此靠很近的物體時可能比較困難。
  • 4. Bounding box、confidence 與 class probability 每個 grid cell 會預測 bounding box 的位置與大小，並估計 confidence score。 同時，模型也會預測該區域可能屬於哪些類別。最後會將位置、是否有物體、 框的重疊程度和類別機率整合，形成每個候選框的分數。
  • 5. NMS 篩選最終結果 模型一開始可能會產生很多重疊的框。NMS 會根據分數和框之間的重疊程度， 移除重複性太高的框，保留最能代表該物體的位置。最後畫面上才會只剩下較乾淨的偵測結果。
Task 05 · 應用案例

YOLO 的實際應用情境：醫學影像輔助偵測

我選擇醫學影像作為應用案例，因為它很適合說明 YOLO 在「快速找出可能目標位置」上的價值， 也能看出物件偵測演算法在高風險場景中不能只追求速度。

案例說明

在醫學影像中，YOLO 可以用來協助標記 X 光片、超音波、內視鏡影像或顯微影像中的可疑目標。 例如在大量影像中先找出疑似病灶、異常組織、細胞型態變化，或是醫療器材的位置。 它的角色比較像是「初步篩選與提示工具」：先把可能需要注意的地方框出來， 讓專業人員可以更有效率地檢查，而不是從零開始逐張影像尋找。

Pros

  • 可以快速處理大量影像，適合用在初步篩選或輔助標記。
  • 輸出結果是 bounding box，判讀者能直接看到模型注意的位置。
  • 如果資料集標註品質良好，可以協助降低重複性檢查的負擔。
  • 能和人工判讀搭配，讓醫師或研究人員優先檢查高風險區域。

Cons

  • 醫學影像的資料標註成本高，如果標註標準不一致，模型學到的判斷也會受影響。
  • 不同醫院、不同儀器、不同影像條件可能造成 domain shift，模型在新資料上表現可能下降。
  • 有些病灶不一定有清楚邊界，用 bounding box 框選時可能無法完整呈現病理特徵。
  • 醫學情境重視可解釋性，只有框出位置仍不足以說明診斷依據，因此需要專業人員複核。
Task 06 · 學習反思

學習 YOLO 後的反思

這一段整理我從初步認識 YOLO，到查資料、比對內容、修正理解後，最後真正學到的成果。

GPT 對學習的幫助

GPT 對我最大的幫助不是直接給答案，而是把我看文章時卡住的地方換一種說法解釋。 像 grid cell 負責物體中心點、confidence score 和 IoU 的關係， 這些概念如果只看文章，很容易看過去但沒有真的吸收。

GPT 回答的限制

GPT 有時會把複雜概念講得比較順，但也可能因此讓人忽略細節。 例如 one-stage detector、confidence score、mAP 這些詞，如果只接受簡化版說明， 就很容易少掉資料集、評估指標和模型限制的脈絡。

我最後真正學到的成果

我最後學到的是，YOLO 不是單純「把物體框起來」這麼簡單， 而是一套把影像特徵、位置預測、類別判斷和結果篩選整合在一起的偵測流程。

一開始我對 YOLO 的理解其實很粗略，只知道它常被拿來做即時物件偵測， 也知道它可以在圖片中畫出框線。但真正開始整理這份網頁後， 才發現自己對許多中間步驟並不熟悉。例如 grid cell 為什麼要負責物體中心點、 bounding box 的 confidence score 到底代表什麼、NMS 為什麼可以刪掉重複框， 以及 mAP 為什麼比單純 accuracy 更適合評估物件偵測模型。

在準備這份筆記的過程中，我先閱讀他人的 YOLO 筆記與教學文章， 再把看不懂的地方拿去問 GPT。GPT 可以快速把觀念講成比較白話的版本， 但我也發現不能只停在這一步，因為有些回答雖然聽起來合理， 還是需要回頭對照文章內容、公式定義與流程圖。這個過程讓我從「大概知道 YOLO 是什麼」， 慢慢變成能說出它從輸入影像到輸出偵測框的完整邏輯。

最後整理成網頁時，我也重新思考哪些內容才是學習 YOLO 時真正重要的重點。 對我來說，這份作業不是把資料複製到網頁上，而是把分散在不同網站的內容重新消化： 哪些是 YOLO 的核心概念、哪些是我容易誤會的地方、哪些需要查證後再修正。 經過這個過程後，我比較能用自己的話說明 YOLO 的基本架構， 也更能理解物件偵測為什麼需要同時考慮速度、定位準確度、分類能力和應用場景。
